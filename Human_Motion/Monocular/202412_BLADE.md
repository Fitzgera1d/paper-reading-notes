## 一、问题定义与研究目标

BLADE 研究的是单张图像人体网格恢复（single-image human mesh recovery, HMR）：给定一张 in-the-wild 图像，恢复图中人物的 3D 人体网格、人体姿态与形状参数，同时恢复能够把 3D mesh 正确投影回图像的透视相机参数。论文关注的不是远距离、近似正交投影下的常规 HMR，而是 close-range 图像中的强透视畸变场景，例如自拍视频、视频会议、近距离手臂伸向镜头、脚靠近镜头等情况。

**核心任务是什么**：从单视角 RGB 图像中估计 SMPL-X 人体模型参数和完整透视投影参数，使输出的 3D 人体几何既在 3D 指标上准确，又能在 2D 图像中与人物轮廓对齐。论文的核心变量包括人体形状 $\beta$、姿态 $\theta$、pelvis 深度 $T_z$、焦距 $f$ 和 3D 平移 $T=[T_x,T_y,T_z]$。

**输入（Input）**：主输入是一张 RGB 图像 $I$。pipeline 同时使用目标人物的 bounding box crop $I_{crop}$ 来估计 pelvis 深度 $T_z$，并使用完整图像 $I$ 进行姿态与形状估计。根据 Figure 4，depth estimator 的输入是目标人物 crop，pose estimator 的输入是完整图像和预测的 $T_z$ 条件。

**输出（Output）**：方法输出 SMPL-X 形状与姿态参数 $(\beta,\theta)$，其中 $\beta\in\mathbb{R}^{10}$，$\theta\in\mathbb{R}^{3K}$，$K=54$ 表示 SMPL-X 关节数；SMPL-X 函数 $M(\beta,\theta)$ 生成 mesh $M\in\mathbb{R}^{10475\times3}$ 和 joints $J\in\mathbb{R}^{54\times3}$。论文明确说明由于 close-range 数据缺少相关标签，本文省略 facial expressions 和 hand gestures。相机输出为焦距 $f$ 和 3D 平移 $T=[T_x,T_y,T_z]$，最终通过透视投影把 mesh 渲染到图像平面。

**任务的主要应用场景**：单张照片或互联网图像的人体动作捕捉、视频会议中的近距离人体重建、虚拟试衣、健康与健身分析、人机交互、内容生产中的人物 mesh 标注，以及大规模图像数据的人体姿态与相机参数自动标注。

**当前任务的关键挑战**：第一，单张图像同时估计人体形状、姿态、尺度、相机内参和外参，本身是欠约束问题，存在尺度歧义和姿态歧义。第二，in-the-wild 图像通常没有相机内参和人物真实尺度。第三，许多 HMR 方法使用 weak-perspective 或 near-orthographic 假设，把 2D affine scale 和 translation 启发式转换成焦距与 3D 平移，这在远距离图像中尚可近似，但在 close-up 图像中会严重错误。第四，现有方法常在 3D pose 准确性和 2D alignment 之间做 trade-off：如果相机模型错误，即使 mesh 在 3D 中看似合理，也无法准确贴合图像轮廓；若强行追求 2D 对齐，又可能牺牲真实 3D 姿态。

**本文主要针对哪些挑战提出改进**：BLADE 的核心目标是摆脱从正交相机参数启发式推导透视相机参数的做法。作者把问题拆成更可解的三个子问题：先估计与透视畸变强相关的 pelvis 深度 $T_z$，再用 $T_z$ 条件化人体姿态与形状估计，最后在已知 $T_z$ 和 mesh 的情况下把 $f,T_x,T_y$ 视为 2D alignment 参数并通过可微渲染优化求解。

## 二、核心思想与主要贡献

论文的直观动机是：close-range HMR 中最关键的透视畸变不是由焦距本身决定，而主要由人物相对相机的 Z 方向距离 $T_z$ 决定。根据透视投影公式，焦距 $f$ 对图像坐标只起全局线性缩放作用；相反，$T_z$ 进入分母 $z_m+T_z$，会非线性改变身体不同部位的相对投影尺度。因此，与其直接从单图同时回归所有相机参数，不如先把 $T_z$ 从其他变量中解耦出来，并把 $T_z$ 当作 close-range pose estimation 的条件信息。

与 BLADE 最相关的已有工作包括 SPEC、ZOLLY、TokenHMR、CLIFF、AiOS 和 SMPLer-X。SPEC 尝试直接估计相机相关参数；ZOLLY 估计深度和正交 affine transformation，再用启发式公式转换成透视焦距和平移；TokenHMR 分析 near-orthographic 相机假设对 HMR 的影响，并通过 loss scaling 缓解 2D/3D trade-off；AiOS 是强大的 whole-body mesh recovery backbone，但其训练分布主要偏向远距离人物，不能天然处理 close-range 强透视畸变。BLADE 的关键差异是：它不从正交参数反推透视相机，而是显式预测 $T_z$，用 $T_z$ 条件化 mesh recovery，并通过 mask alignment 解出剩余投影参数。

本文最重要的贡献可以概括为三点。第一，提出一个无需正交相机启发式假设的单图 HMR pipeline，直接恢复完整透视投影模型中的关键参数。第二，指出 close-range pose estimation 受 $T_z$ 强影响，并设计 ControlNet-style 的 $T_z$-conditioned pose estimator，在保留 AiOS 泛化能力的同时学习近距透视畸变对人体外观和 3D 姿态的影响。第三，构建 Bedlam-CC 合成数据集，提供大量 close-range、带准确 SMPL-X 和相机标签的图像，用于补足现有 HMR 数据在近距离深度分布上的缺口。

## 三、方法与实现细节（全文重点）

本节除特别说明外均依据论文正文与补充材料。官方代码已经公开，但本笔记没有做代码级补查；正文只保留论文中足以支撑方法理解的细节，复现层面的缺口集中放在 3.6 末尾。

### 3.1 整体 Pipeline 概述

BLADE 的 pipeline 可以理解为“先解耦深度，再估计人体，再求相机对齐”的三阶段流程。第一阶段，Pelvis Depth Estimator $F^{T_z}$ 从目标人物 crop $I_{crop}$ 中预测 pelvis 到相机的 Z 方向距离 $T_z$。第二阶段，$T_z$-aware Pose Estimator $F^{pose}$ 以完整图像 $I$ 和 $T_z$ 为条件，预测 SMPL-X 形状与姿态 $(\beta,\theta)$，并通过 SMPL-X 得到 vertices $V$ 和 joints $J$。第三阶段，Camera Solver 在固定或可选微调 $T_z$、mesh 和 global orientation 的条件下，通过可微 rasterization 把预测 mesh 渲染成 mask，并最大化该 mask 与人物分割 mask 的 IoU，从而求解焦距 $f$ 和平移 $T_x,T_y$。训练阶段只训练 depth estimator 和 pose estimator；camera solver 是 test-time differentiable optimization，不需要学习参数。

### 3.3 关键模块逐个拆解

#### 3.3.1 透视几何分解

BLADE 的方法建立在一个简单但关键的透视投影分析上。SMPL-X 输出的 mesh 顶点坐标记为 $[x_m,y_m,z_m]$，人体 pelvis 在相机坐标系下的位置为 $T=[T_x,T_y,T_z]$，则相机空间坐标为：

$$
\begin{bmatrix}x,y,z\end{bmatrix}=\begin{bmatrix}x_m,y_m,z_m\end{bmatrix}+\begin{bmatrix}T_x,T_y,T_z\end{bmatrix}.
$$

透视投影为：

$$
\begin{bmatrix}u\\v\end{bmatrix}=f\cdot\begin{bmatrix}x/z\\y/z\end{bmatrix}=f\cdot\begin{bmatrix}(x_m+T_x)/(z_m+T_z)\\(y_m+T_y)/(z_m+T_z)\end{bmatrix}.
$$

从这个公式可以看到，$f$ 乘在整体投影坐标外面，主要控制图像尺度；$T_z$ 出现在分母中，会改变身体不同深度处顶点的相对投影比例。也就是说，改变焦距像是 zoom，改变 $T_z$ 才会改变 close-range perspective distortion。论文用 Figure 3 说明：同一个人物在不同焦距下可以通过 crop/zoom 得到相同的畸变形态，但当 $T_z$ 变小时，近端肢体会被明显放大，透视畸变非线性增强。

这个分析带来两个设计后果。第一，$T_z$ 可以被当作从图像外观中可学习的关键变量，因为 close-up distortion 本身提供了深度线索。第二，人体姿态估计器不能忽略 $T_z$，因为同一 3D pose 在不同 $T_z$ 下呈现的 2D 外观显著不同；把 close-up 图像当成普通远距离图像会导致错误的 3D pose。

#### 3.3.2 Pelvis Depth Estimator $F^{T_z}$

Pelvis Depth Estimator 的输入是目标人物 crop $I_{crop}$，输出是 pelvis depth $T_z=F^{T_z}(I_{crop})$。论文选择 Depth Anything V2 作为预训练 monocular depth backbone，并在 Figure 4 中画出其内部特征流：`Patch Embed -> Transformer -> DPT Head`，随后接一个 learnable ConvNet 和 $T_z$ Head 输出 scalar depth。论文明确说明在 DINOv2、Sapiens 和 DAv2 等 backbone 中，DAv2 对该任务效果最好。

该模块解决的是“先从所有相机变量中抓住最影响透视畸变的变量”。由于深度范围理论上无上界，直接在所有距离上均匀追求绝对误差并不合理；作者强调 close-range，尤其 $T_z<1.2$m 时，透视畸变变化更强，对 HMR 更关键。因此 depth loss 对近距离样本赋予更高权重：

$$
L_{depth}=\frac{1}{T_z^{GT}}\left\Vert T_z-T_z^{GT}\right\Vert_1.
$$

这个 loss 的物理含义是：同样的米级误差，在近距离下会造成更大的投影畸变变化，因此训练时应更重视近距离 depth accuracy。消融结果也支持这一设计方向：在 SPEC-MTP 上，DINOv2、Sapiens、DAv2 和最终配置的 $E_{T_z}$ 分别为 0.300m、0.210m、0.154m 和 0.127m，说明 DAv2 特征与 Bedlam-CC/augmentation 对近距 depth estimation 有明显贡献。

#### 3.3.3 $T_z$-aware Pose Estimator $F^{pose}$

Pose estimator 的目标是在完整图像 $I$ 和预测深度 $T_z$ 条件下输出 SMPL-X 参数：

$$
(\beta,\theta)=F^{pose}(I\mid T_z),\qquad (V,J)=M(\beta,\theta).
$$

BLADE 以 AiOS 作为 HMR backbone。AiOS 的优点是直接从原始未裁剪图像中预测人体 mesh，包含 transformer-based encoder 和 non-autoregressive decoder，用于 set prediction。问题是 AiOS 的训练数据主要包含远距离人物，因此对 close-range perspective distortion 不够适应。作者观察到，直接用 HuMMan、PDHuman 和 H36M 等较小/偏分布数据 fine-tune AiOS 会过拟合并损害真实 close-range 数据上的泛化。

为了解决这个问题，BLADE 使用 ControlNet-style 的结构：保留一个 frozen AiOS branch，另建一个 trainable copy branch。trainable copy 用预训练 AiOS 权重初始化，其输出先经过 zero-initialized MLP，再与 frozen branch 的输出相加。zero-MLP 的作用很关键：训练刚开始时 residual 为 0，整体模型退化为原始 AiOS，避免一开始破坏已有泛化能力；随着训练进行，residual 逐渐变为非零，copy branch 学到 close-range 条件下的修正。

$T_z$ 条件注入通过两个 MLP 完成：先把 scalar $T_z$ 编码为 deep features，再把这些 features 加到 trainable backbone 的 encoder features 中。这样 frozen branch 负责保留原始视觉与人体先验，trainable branch 负责学习“这个 depth 下图像畸变应该如何影响 3D pose 和 shape”。从 Figure 4 看，`T_z Encoder` 输入 trainable copy，并通过 residual fusion 进入 pose head。

消融实验说明该模块不是简单 fine-tuning 的替代品，而是有实质贡献。在 SPEC-MTP 上，raw AiOS 的 PA-MPJPE/MPJPE/PVE 为 62.816/101.577/110.851，直接 fine-tune AiOS 后变为 64.932/113.173/120.582，反而更差；BLADE 的 $T_z$ conditioning 达到 56.666/94.050/99.635，说明它同时保留了泛化能力并利用了 depth condition。

#### 3.3.4 Camera Solver：从 $T_z$ 和 mesh 求解 $f,T_x,T_y$

在估计了 $T_z$ 与 mesh 后，BLADE 把剩余相机参数 $f,T_x,T_y$ 视为 image-space alignment 变量，而不是继续用神经网络直接回归。初始化时，使用 $T^{init}=[0,0,T_z]$ 和 $f^{init}=h$，其中 $h$ 是图像高度。然后将预测 SMPL-X mesh rasterize 成二值 mask：投影到 mesh surface 的像素为 1，其余为 0。

为了获得 2D 对齐目标，作者使用 off-the-shelf MediaPipe 对人物进行分割，得到 segmentation mask。Camera Solver 通过可微 rasterization 优化 $(f,T_x,T_y)$，最大化渲染 mesh mask 与 segmentation mask 的 IoU。为了让梯度在整个图像上更平滑地传播，作者对 rasterized mask 和 segmented mask 都做 Gaussian smoothing。

这个设计的意义是：当 $T_z$ 固定时，$T_x,T_y$ 主要控制 $z=T_z$ 平面内的平移，$f$ 主要控制整体缩放，三者确实更像 2D alignment 参数。相比从 weak-perspective scale 启发式反推焦距，该优化直接以图像轮廓对齐为目标，因此能在 close-up 图像中得到更可靠的投影参数。

论文还指出，额外优化 $T_z$，以及可能优化 pose 和 global orientation，通常会进一步提升 pose 和 camera parameters 的质量。这里的关键不是把相机参数从弱透视公式中直接换算出来，而是把相机恢复变成一个由预测 mesh、预测深度和图像 mask 共同约束的 alignment problem。

### 3.4 损失函数与训练目标

BLADE 有两个需要训练的模块：depth estimator 和 pose estimator。Camera Solver 是优化过程，不需要训练。

**Depth loss** 监督预测 pelvis depth：

$$
L_{depth}=\frac{1}{T_z^{GT}}\left\Vert T_z-T_z^{GT}\right\Vert_1.
$$

其中 $T_z^{GT}$ 是 ground-truth pelvis depth。$\frac{1}{T_z^{GT}}$ 使近距离样本权重更高，因为 $T_z$ 越小，透视畸变对投影越敏感。

**Shape loss** 监督 SMPL-X shape coefficients：

$$
L_{shape}=L_1(\beta,\beta^{GT}).
$$

其中 $\beta\in\mathbb{R}^{10}$ 是预测 shape，$\beta^{GT}$ 是 ground-truth shape。

**Pose loss** 使用 joint rotation 的 angular error：

$$
L_{pose}=E_{ang}(\theta,\theta^{GT}).
$$

其中 $\theta$ 包含 root joint orientation 和各关节旋转。$E_{ang}$ 表示旋转之间的角度误差，用于直接约束关节旋转。

**Joint loss** 监督 SMPL-X joints：

$$
L_{joint}=L_1(J,J_{GT}).
$$

其中 $J\in\mathbb{R}^{54\times3}$ 来自 SMPL-X 输出。

**Vertex loss** 监督 mesh vertices：

$$
L_{vert}=L_1(V,V_{GT}).
$$

其中 $V\in\mathbb{R}^{10475\times3}$ 是预测 mesh vertices。

**Pose estimator 总损失** 为：

$$
L=w_{shape}L_{shape}+w_{pose}L_{pose}+w_{joint}L_{joint}+w_{vert}L_{vert}.
$$

论文给出的权重为 $w_{shape}=1$、$w_{pose}=1$、$w_{joint}=5$、$w_{vert}=5$。这些权重用于平衡不同 loss 的数量级。消融表没有单独移除各 loss 项，而是重点验证 depth backbone 和 $T_z$ conditioning 的贡献。

**Camera Solver objective** 没有作为训练 loss，而是在推理时优化：

$$
\max_{f,T_x,T_y}\ \operatorname{IoU}\left(\operatorname{Rasterize}(M(\beta,\theta),f,[T_x,T_y,T_z]),\operatorname{SegMask}(I)\right).
$$

这是根据论文文字对目标的结构化表达：[Inference]。论文明确写出优化目标是最大化 rasterized SMPL-X mask 与 segmentation mask 的 IoU。

### 3.5 数据集与数据处理

**训练数据**：论文使用 ZOLLY 中采用的 3D 数据子集，包括 H36M、PDHuman 和 HuMMan。这些数据提供 camera 和 SMPL 参数，作者使用 SMPL-X 论文中的转换方法把标签转换到 SMPL-X。Depth estimator 还使用作者新构建的 Bedlam-CC 数据，以补充近距离透视畸变样本。为保证 pose estimation 对比公平，论文明确说明 Bedlam-CC 不用于 pose learning。

**Bedlam-CC**：这是本文新增的大规模 synthetic dataset，全称中的 “CC” 表示 close camera。它基于 BEDLAM assets 生成，总量为 2 million synthetic images。主文描述中，作者按 inverse depth $1/T_z$ 近似透视畸变曲线采样，强制 80% 样本落在 $0.3\text{m}\leq T_z\leq1.2\text{m}$，其余样本落在 $1.2\text{m}<T_z\leq10\text{m}$。补充材料在 evaluation split 描述中写的是 80% 落在 $0.5\text{m}\leq T_z\leq1.2\text{m}$，剩余落在 $1.2\text{m}<T_z\leq10\text{m}$，并说明从 2 million generated images 中取 1314 张作为 evaluation split。这里存在主文与补充材料表述范围下界不同的问题，本笔记保留该差异，不合并成单一断言。

**Bedlam-CC 生成细节**：补充材料说明，camera position 位于以 $T_z$ 为半径的球面上，azimuth $\theta\in[0,2\pi]$，elevation $\phi\in[0.1\pi,0.7\pi]$。相机朝向通过 LookAt() 指向 SMPL-X spine 上随机 target bone，并加入随机噪声 $v\in\mathbb{R}^3$。焦距通过 dolly zoom 设置，默认 1m 距离下 $f_d=15$mm，sensor size 为 $36\times36$mm，并随机采样 $f_{GT}\in[0.7,1.3]\cdot f_d$。lighting 包括 skylight 的 background/intensity，以及 directional sun light 的 position/color/intensity 随机化。由于数据通过 SMPL-X 和 Unreal Engine 生成，不需要从其他人体模型转换到 SMPL-X，从而避免了 PDHuman 中类似的 conversion error。

**评测数据**：主要评测包括 SPEC-MTP、PDHuman、Bedlam-CC 和 HuMMan。SPEC-MTP 是 real-world smartphone capture，距离约 0.5m 到 2m，多数样本约 1m，适合 close-range 评测。PDHuman 是 synthetic dataset，距离约 0.5m 到 1.8m，很多样本约 0.6m，但补充材料指出其部分 pose labels 与图像不对齐，因此 quantitative results 不一定完全代表真实效果。HuMMan 是 studio multi-view 数据，视觉多样性有限，$T_z$ 主要集中在约 1.9m，不能充分测试 close-range distortion。Bedlam-CC 被用来提供更准确、分布更宽的 close-range benchmark。

**数据处理**：depth estimator 使用人物 crop；pose estimator 使用完整图像；camera solver 使用 MediaPipe 分割的人物 mask 和 differentiable rasterization 得到的 mesh mask。depth backbone 消融中提到最终配置使用 improved augmentation 和 additional Bedlam-CC data，说明近距离数据分布和增强策略直接影响 $T_z$ 估计质量。

### 3.6 训练流程、推理流程与复现备注

**训练流程**：BLADE 分两阶段训练。第一阶段训练 pelvis depth estimator $F^{T_z}$，总 batch size 为 128，使用 8 张 NVIDIA A100 GPU，训练 4 epochs。第二阶段冻结 $F^{T_z}$，把它预测的 $T_z$ 输入到 pose estimator $F^{pose}$，训练 $T_z$-conditioned pose estimator；该阶段 batch size 为 336，使用 48 张 NVIDIA A100 GPU，训练 4 epochs。Camera Solver 中的 $f,T_x,T_y$ 优化没有学习参数，因此不需要训练。

**推理流程**：给定图像 $I$ 和目标人物 crop $I_{crop}$，先用 $F^{T_z}$ 预测 pelvis depth $T_z$；然后把 $I$ 与 $T_z$ 输入 $F^{pose}$，得到 $(\beta,\theta)$，并通过 SMPL-X 得到 mesh $V$；接着用 $T^{init}=[0,0,T_z]$ 和 $f^{init}=h$ 初始化投影，将 mesh rasterize 成 mask；同时用 MediaPipe 得到人物 segmentation mask；最后通过 differentiable rasterization 优化 $f,T_x,T_y$，得到最终的 full perspective projection model 和 mesh rendering。论文还提到可选优化 $T_z$、pose 和 global orientation 会改善结果。

**训练与推理差异**：训练阶段使用 ground-truth camera/SMPL-X 标签监督 depth 与 pose；推理阶段不依赖 ground-truth camera，而是先预测 $T_z$，再用 mask alignment 解 $f,T_x,T_y$。Bedlam-CC 用于 depth estimator 训练和评测，但不用于 pose estimator 训练。

**复现备注**：官方代码仓库已公开，但本笔记没有进行代码级补查。论文正文、补充材料和 TeX 源已经足够确认方法主线、几何公式、损失、训练阶段、数据集和核心实验。若要完整复现，仍有几类高价值细节需要查代码确认：depth estimator 中 DAv2 后接 ConvNet / $T_z$ head 的具体结构；$T_z$ condition 注入 AiOS trainable copy 的具体层位与 MLP 配置；camera solver 的优化器、迭代次数、Gaussian smoothing 参数，以及可选优化 $T_z$/pose/global orientation 的默认开关。

## 四、实验结果与有效性说明

论文的实验重点不是只比较 3D mesh error，而是同时评估 3D 几何、2D alignment 和透视投影参数。评测指标包括 PVE（mesh 顶点误差，mm，越低越好）、mIoU（渲染 mask 与图像 mask 的平均 IoU，越高越好）、$E_f=|f_{pred}-f_{GT}|/f_{GT}$、$E_{T_{xy}}=\Vert T_{xy}^{pred}-T_{xy}^{GT}\Vert_2$、$E_{T_z}=|T_z^{pred}-T_z^{GT}|$，以及反深度误差 $E_{1/T_z}=|1/T_z^{pred}-1/T_z^{GT}|$。$E_{1/T_z}$ 的引入符合论文的几何分析：远距离下同样的 $T_z$ 误差对透视畸变影响较小，反深度能更强调 close-range 区域。

在主表的 SPEC-MTP real-world close-range 数据上，`Ours (real-world)` 达到 $E_{T_z}=0.127$m、$E_{1/T_z}=0.112$、$E_{T_{xy}}=0.044$m、$E_f=0.159$、PVE 99.6mm、mIoU 69.5。对比 ZOLLY 的 $E_{T_z}=0.899$m、$E_f=1.063$、PVE 126.7mm、mIoU 62.3，BLADE 在 depth、camera、mesh 和 alignment 上同时更好。论文文字总结为：在 SPEC-MTP 上，相比 close-range SOTA ZOLLY，BLADE 的 $E_{T_z}$ 相对提升 85.9%，PVE 相对提升 21.4%。

在 PDHuman 上，`Ours` 达到 $E_{T_z}=0.106$m、$E_f=0.216$、PVE 80.5mm、mIoU 67.3，是主表中最好的整体结果之一。但作者也谨慎指出，PDHuman 的 synthetic domain 和标签不一致问题会影响其代表性，因此他们额外报告 `Ours (real-world)`，通过增加 Human3.6M 采样、减少 PDHuman 采样来更适配真实图像。在 Bedlam-CC 上，`Ours (real-world)` 达到 $E_{T_z}=0.325$m、$E_f=0.212$、PVE 106.8mm、mIoU 75.0，说明其在更准确的 synthetic close-range benchmark 上也保持优势。

Depth backbone 消融证明 $T_z$ estimation 是整条 pipeline 的关键。DINOv2、Sapiens、DAv2 和最终配置在 SPEC-MTP 上的 $E_{T_z}$ 分别为 0.300m、0.210m、0.154m 和 0.127m。作者的解释是，DAv2 作为 monocular depth foundation model 更适合该任务，而额外的 Bedlam-CC close-range 数据和改进 augmentation 进一步提升近距离 depth accuracy。

$T_z$ conditioning 消融证明，BLADE 的 pose estimator 不是简单 fine-tune AiOS。raw AiOS 在 SPEC-MTP 上的 PA-MPJPE/MPJPE/PVE 为 62.816/101.577/110.851；直接 fine-tune 后反而退化为 64.932/113.173/120.582；使用 ControlNet-style $T_z$ conditioning 后提升到 56.666/94.050/99.635。这说明 close-range 问题需要在保留原模型知识的同时注入 depth-aware residual，而不是用小规模 close-range 数据直接覆盖 backbone。

可视化结果主要说明 BLADE 在靠近镜头的手、脚、腿部等区域更能匹配强透视畸变。AiOS 和 ZOLLY 常出现人物距离估计不准、肢体深度关系错误、mesh 与轮廓错位等问题；BLADE 因为先估 $T_z$ 并用 mask alignment 解投影参数，最终 rendering 与图像人物边界更一致。

局限性需要克制看待。第一，单图 HMR 的尺度和姿态歧义没有被完全消除，BLADE 只是利用人体尺度先验、深度估计和 mask alignment 把问题变得更可解。第二，camera solver 依赖人物 segmentation mask，若 MediaPipe 分割失败，alignment 优化可能受到影响。第三，论文未详细公开 camera solver 的优化超参数和若干网络层级细节，如果要复现，需要进一步查官方代码。第四，方法主要输出 SMPL-X body，不建模 facial expressions 和 hand gestures，这会限制全身细节表达。第五，Bedlam-CC 是 synthetic 数据，虽然标签准确且 close-range 分布更好，但真实世界材质、遮挡和复杂背景仍可能带来 domain gap。
